# tree_analytics.py

Строит сводные гистограммы по всем генам сразу: сколько клад, какого они размера, какой глубины, и какая генетическая дистанция от предка до листьев.

## Входные и выходные файлы

Все пути определяются не внутри самого скрипта, а централизованно — в файле **`paths.py`** (лежит в `scripts/`).

| Путь | Где физически | Что это |
|---|---|---|
| входные данные | `data/<key>/clades_report.json` | JSON-отчёт с кладами (можно переопределить имя файла флагом `--input-file`) |
| результаты | `results/<key>/tree_analytics.png` | сохранённый график |

Ключ передаётся аргументом `-k` / `--key` при запуске.

## Задача

Скрипт читает JSON-отчёт (формат — словарь `{ген: {"mrbayes": {"clades": [...]}}}`) и для каждой клады забирает уже готовые (не пересчитывает сам) поля:
- `size` — сколько листьев в кладе;
- `depth` — глубина клады;
- `ancestor_to_leaves.mean` — средняя генетическая дистанция (замен на сайт) от предка клады до её листьев.

Дальше строит 4 графика рядом:
1. Количество клад по каждому гену.
2. Распределение размера клады (`size`).
3. Распределение глубины клады (`depth`).
4. Распределение средней дистанции предок→листья.

## Важное про `ancestor_to_leaves`

Это **не время** в буквальном смысле, а генетическая дистанция — число замен на сайт (substitutions/site) на пути от общего предка клады до конкретного листа. Чем больше значение — тем сильнее разошлись последовательности внутри клады. В единицах реального времени это можно интерпретировать только если дерево дополнительно откалибровано по времени (у нас — нет, это обычное filogenetic-дерево от mrbayes/iqtree).

## Запуск

```bash
python3 tree_analytics.py --key <название_подпапки>
```

Например:
```bash
python3 tree_analytics.py --key tree_demo
```

Если входной файл называется не `clades_report.json`, а как-то иначе:
```bash
python3 tree_analytics.py --key tree_demo --input-file report3.json
```
