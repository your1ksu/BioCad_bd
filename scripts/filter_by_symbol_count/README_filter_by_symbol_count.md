# filter_by_symbol_count.py

Отбирает fasta-файлы по количеству последовательностей внутри (считает символы `>` — каждый `>` это начало одного FASTA-заголовка, то есть одна последовательность).

## Входные и выходные файлы

Все пути определяются не внутри самого скрипта, а централизованно — в файле **`paths.py`** (лежит в `scripts/`).

`grouped_by_germlines/vj/` — это результат предыдущего шага пайплайна (`group_by_germlines.py`), поэтому вход берётся **из `results/`**, а не из `data/`.

| Путь | Где физически | Что это |
|---|---|---|
| входные данные | `results/<key>/grouped_by_germlines/vj/` | fasta-файлы, сгруппированные по паре V+J генов |
| результаты | `results/<key>/vj_filtered/` | сюда копируются только прошедшие фильтр файлы (на уровне `results/<key>/`, рядом с самой папкой `grouped_by_germlines`, а не внутри неё) |

Ключ передаётся аргументом `-k` / `--key`. **По умолчанию — `BCR`.**
Цепочка подпапок передаётся аргументом `--subfolders`. **По умолчанию — `grouped_by_germlines vj`.**

## Задача

Для каждого fasta-файла в целевой папке скрипт считает, сколько раз встречается символ `>` (то есть сколько последовательностей в файле), и копирует в новую папку `<последняя_подпапка>_filtered` только те файлы, где это число:
- **строго больше 5**
- **строго меньше 100**

Слишком маленькие группы (≤5 последовательностей) и слишком большие (≥100) отсеиваются — остаются только "средние по размеру" гены, наиболее подходящие для дальнейшего анализа (например, построения дерева).

Новая папка создаётся на уровне `results/<key>/` — то есть рядом с самой папкой `grouped_by_germlines`, а не внутри неё.

## Запуск

```bash
python3 filter_by_symbol_count.py
```
(по умолчанию обработает `results/BCR/grouped_by_germlines/vj/` → сохранит в `results/BCR/vj_filtered/`)

Другой ключ или другая цепочка подпапок:
```bash
python3 filter_by_symbol_count.py --key BCR --subfolders grouped_by_germlines d
```
