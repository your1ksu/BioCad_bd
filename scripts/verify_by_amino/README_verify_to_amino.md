# verify_by_amino.py

Скрипт валидирует нуклеотидные fasta-последовательности (`V+J`, выровненные по гермлайну). Он транслирует их в аминокислоты для проверки рамок считывания, стоп-кодонов и сверки со справочником IMGT, но на выходе сохраняет исходные нуклеотиды.

## Входные и выходные файлы

Все пути определяются не внутри самого скрипта, а централизованно — в файле **`paths.py`** (лежит в `scripts/`).

| Путь | Где физически | Что это |
|---|---|---|
| входные данные | `results/<key>/vj_filtered/` | nt-fasta файлы (результат `filter_by_symbol_count.py`) |
| результаты | `results/<key>/verify_by_amino/` | успешно транслированные и подтверждённые аминокислотные fasta (заголовки сохраняются как в исходном файле) |
| преждевременный стоп | `results/<key>/verify_by_amino/premature_stop/` | последовательности, где ни в одной из 3 рамок стоп-кодон не оказался в конце |
| нет совпадения | `results/<key>/verify_by_amino/no_match/` | последовательности, прошедшие проверку по стоп-кодону, но не нашедшие похожего гена в справочнике ни в одной рамке |
| IMGT AA-справочник | `data/references/HomoSapiens_IMGTGENEDB-ReferenceSequences.fasta` | подаётся автоматически, нужно скачать вручную (см. ниже) |

Ключ передаётся аргументом `-k` / `--key`. **По умолчанию — `BCR`.**

### Откуда взять AA-справочник

Ссылка (аминокислотные последовательности, без IMGT-гэпов, только функциональные+ORF+in-frame псевдогены):
```
https://www.imgt.org/download/GENE-DB/IMGTGENEDB-ReferenceSequences.fasta-AA-WithoutGaps-F+ORF+inframeP
```
IMGT блокирует автоматическое скачивание (robots.txt) — открой ссылку в браузере вручную, сохрани файл и положи по пути выше под именем `aa_reference_noGaps.fasta`.

Если справочника нет по этому пути — скрипт не упадёт, а просто пропустит сверку и примет первый вариант, прошедший проверку по стоп-кодону (с предупреждением в консоли).

## Задача

Для каждой нуклеотидной последовательности:

**1. Рамка считывания.** Пробуем рамки по очереди: 1 → 2 → 3 (это позиции сдвига 0, 1, 2 нуклеотида). Обычно подходит рамка 1, но не всегда — если она не даёт валидного результата, пробуем следующую.

**2. Обрезка "хвоста".** Если после сдвига на рамку в конце остаётся 1-2 "лишних" нуклеотида, не складывающихся в целый кодон — они просто отбрасываются (не транслируются).

**3. Стоп-кодон.** Транслируем через `Bio.Seq.translate()` без обрезки по первому стоп-кодону, чтобы видеть все стоп-кодоны сразу:
   - стоп-кодона нет вообще → рамка валидна;
   - стоп-кодон есть, ровно один, и он в самом конце → рамка валидна, символ `*` убирается из финального белка;
   - стоп-кодон встретился раньше конца (или их несколько) → рамка невалидна, пробуем следующую.

**4. Сверка со справочником.** Валидный (по стоп-кодону) вариант белка сравнивается со всеми генами AA-справочника методом локального попарного выравнивания (тот же принцип, что и в `group_by_germlines.py`). Совпадением считается покрытие ≥50% длины найденного гена (константа `MATCH_IDENTITY_THRESHOLD` в начале файла — можно поднастроить). Если совпадение нашлось — на этом останавливаемся и берём этот вариант. Если нет — пробуем следующую рамку.

**5. Если ничего не подошло.** Если ни одна рамка не дала валидный по стоп-кодону результат — последовательность уходит в `premature_stop/`. Если валидный результат был, но совпадения со справочником не нашлось ни в одной рамке — уходит в `no_match/`.

В конце выводится сводка по всем счётчикам — сколько последовательностей попало в каждую категорию, чтобы видеть общее соотношение.

## Запуск

```bash
python3 verify_by_amino.py
```
(по умолчанию: ключ `BCR`, вход `results/BCR/vj_filtered/`, выход `results/BCR/amino/`)

Другой ключ / подпапки / справочник:
```bash
python3 verify_by_amino.py --key BCR --input-subfolder vj_filtered --reference-fasta /путь/к/справочнику.fasta
```
