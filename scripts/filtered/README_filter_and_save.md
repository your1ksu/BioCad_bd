# filter_and_save.py

Фильтрация "сырых" BCR-последовательностей: отсеивает дубликаты и строки с некорректной длиной/локусом.

## Входные и выходные файлы

Все пути (откуда брать данные, куда сохранять результат, где лежат IMGT-справочники) определяются не внутри самого скрипта, а централизованно — в файле **`paths.py`** (лежит здесь же, в `scripts/`).

По одному ключу (названию подпапки) сразу определяются все три пути:

| Путь | Где физически | Что это |
|---|---|---|
| входные данные | `data/<key>/` | сюда кладёшь исходный `.tsv`/`.csv` |
| результаты | `results/<key>/` | сюда пишется отфильтрованный файл |
| IMGT-справочники | `data/IMGT/Homo_sapiens/IG/` | подаётся автоматически, ключ на него не влияет |

Ключ передаётся при запуске:
```bash
python paths.py -k batch1
```
Скрипт использует эту же логику внутри себя (через `get_paths()`), вручную прописывать пути не нужно.

## Задача

Скрипт берёт сырой файл с BCR-последовательностями и отбраковывает "плохие" строки:
- дубликаты по полю `sequence`;
- строки с неизвестным/неподдерживаемым локусом (поддерживаются только `IGH`, `IGK`, `IGL`);
- строки без `v_call`/`j_call` или без `sequence_vdj`;
- строки, где длина `sequence_vdj` не укладывается в ожидаемый диапазон (вычисляется исходя из длины V- и J-гена по IMGT-справочнику + допустимая длина junction-региона).

Для каждой отбракованной строки фиксируется причина — в конце выводится статистика: сколько строк было, сколько осталось, что и почему отсеялось.

**Результат:** отфильтрованный `.tsv` — входной файл для следующего шага (`group_by_germlines.py`).

## Запуск

```bash
python filter_and_save.py
```
