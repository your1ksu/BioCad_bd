# synthetic_data — конструктор синтетических BCR

Генерирует тестовые клональные семейства BCR **из реальных зародышевых
антител** (IMGT germline из `data/references`) с контролируемыми соматическими
мутациями и **точной истиной по регионам** (FR1/CDR1/FR2/CDR2/FR3). Служит для
проверки пайплайна: правильно ли `06_analyze_mutations` относит мутации к
регионам, как ведёт себя разметка в разных условиях.

## Почему из реальных germline, а не random

Каждый предок клона собирается из настоящего гена **V(-D)-J**: берём IMGT-V,
стык CDR3 (у тяжёлых — с обрезком реального D), IMGT-J в правильной рамке
чтения. Последовательность строго от начала **FR1** до конца **FR4**, в рамке,
без стоп-кодонов и **без «мусора» на концах** (нет случайных флангов/адаптеров).
Мутации накладываются поверх — как настоящий SHM.

## Истина по регионам (независимая)

IMGT-нумерация фиксирована: колонки germline с гэпами задают номер кодона, а
номер кодона → регион (FR1 ≤26, CDR1 27–38, FR2 39–55, CDR2 56–65, FR3 66–104,
CDR3 105–117, FR4 ≥118) — те же границы, что `IMGT_DOMAIN_STARTS` в пайплайне.
Поэтому для каждой заложенной мутации регион известен точно и **не зависит** от
IgBLAST — есть с чем сравнивать его разметку. Нонсенс-замены (стоп) не вносятся:
они летальны для B-клетки, продуктивные антитела их не несут.

## Конфиг (`synthetic_config.txt`, текстовый)

| ключ | смысл |
|---|---|
| `groups` | сколько групп (клонов) |
| `sequences` | сколько последовательностей всего (делится по группам) |
| `mutation_rate` | частота мутаций, % длины V-региона на лист |
| `germlines` | `auto` (подобрать разные) или список `V*алл/J*алл` через запятую |
| `chain` | `mix`/`IGH`/`IGK`/`IGL` при `auto` |
| `region_bias` | `none`/`cdr`/`fr` — куда смещать мутации (для проверки регионов) |
| `phylogeny` | `lineage` (ветвление, есть клады) / `star` |
| `mutate_fr4` | мутировать ли также FR4 (J). CDR3-стык не мутируется |
| `seed` | воспроизводимость |

## Запуск

```bash
# 1) сгенерировать датасет
python make_synthetic.py -c synthetic_config.txt -r ../../data/references -o OUT

# 2) превратить FASTA в AIRR-TSV настоящим IgBLAST (как реальные данные)
#    (даёт OUT/BCR_data.tsv для run_pipeline.py -i)

# 3) прогнать пайплайн и проверить разметку регионов
python validate_regions.py --ground-truth OUT/ground_truth.tsv \
    --mutations RESULT/mutation_tables
```

### Выходы `make_synthetic.py`

| файл | что |
|---|---|
| `synthetic_BCR.fasta` | чистые последовательности (вход IgBLAST/пайплайна) |
| `ground_truth.tsv` | по одной строке на кодон-мутацию: регион, IMGT-кодон, замена, синонимичность |
| `groups.tsv` | какой группе какой germline достался |
| `summary.txt` | сводка + эхо конфига |

### Что показывает `validate_regions.py`

1. **imgt_position → region**: согласована ли разметка с IMGT-номером (чистая корректность).
2. **Матрица «истинный регион × определённый»** и общая точность.
3. **Recall по регионам** — сколько заложенных мутаций найдено в верном регионе.

## Разные обстоятельства

Меняя конфиг, смотрим поведение: высокая/низкая `mutation_rate`, смещение
`region_bias=cdr|fr`, `phylogeny=star|lineage`, одна группа vs много, разные
локусы. Так проверяется устойчивость разметки и сборки к условиям.
