# mutation_stats

Локализация соматических мутаций по областям FR/CDR и оценка селективного
давления по отношению dN/dS. Вход — таблица мутаций, формируемая
`scripts/analyze_mutations/analyze_mutations.py`.

## Вход и выход

| | Формат | Источник |
|---|---|---|
| Вход | `mutations.tsv` либо каталог `mutation_tables/` | `analyze_mutations.sh` |
| Выход | таблица в stdout; при `--tsv` — файл TSV | — |

При указании каталога обрабатываются все `mutations.tsv` рекурсивно; результаты
агрегируются, при `--per-clade` дополнительно выводится отчёт по каждой кладе.

Используемые поля входной таблицы: `region`, `ref_codon`, `is_silent`,
`sequence_id`. Строки-заглушки для последовательностей без мутаций (пустое поле
`region`) исключаются.

## Запуск

```
python mutation_stats.py mutation_tables/
python mutation_stats.py mutation_tables/ --per-clade
python mutation_stats.py mutation_tables/clade1/mutations.tsv
python mutation_stats.py mutation_tables/ --tsv stats.tsv
```

Внешние зависимости отсутствуют; требуется Python 3.9 и стандартная библиотека.

## Формат вывода

```
регион                  мутаций    доля     N     S    N/S    dN/dS
------------------------------------------------------------------------------
FR1-IMGT                      3   37.5%     1     2   0.50    0.114
CDR1-IMGT                     5   62.5%     4     1   4.00    2.457
------------------------------------------------------------------------------
ВСЕ CDR                       5   62.5%     4     1   4.00    2.457
ВСЕ FR                        3   37.5%     1     2   0.50    0.114
```

Колонки: `мутаций` — общее число замен области; `доля` — её вклад в суммарное
число замен; `N` и `S` — число несинонимных (`Nd`) и синонимных (`Sd`) замен;
`N/S` — их отношение без нормировки на состав кодонов; `dN/dS` — отношение с
нормировкой (см. «Метод»). Строки `ВСЕ CDR` и `ВСЕ FR` агрегируют области
соответствующего типа.

Прочерк в колонке `dN/dS` означает, что величина не определена: отсутствуют
синонимные замены, отсутствуют сайты, либо достигнуто насыщение (`pS ≥ 3/4`,
поправка Джукса — Кантора не определена).

## Метод

Оценка dN/dS выполняется методом Ней — Годзёбори с поправкой Джукса — Кантора.

### Обозначения

| Величина | Определение | Источник |
|---|---|---|
| `n`, `s` | число несинонимных и синонимных сайтов одного кодона, `n + s = 3` | вычисляется по `ref_codon` |
| `N_sites`, `S_sites` | суммарное число несинонимных и синонимных сайтов области | `Σn`, `Σs` |
| `Nd` | число наблюдаемых несинонимных замен области | строки с `is_silent = no`; в выводе — колонка `N` |
| `Sd` | число наблюдаемых синонимных замен области | строки с `is_silent = yes`; в выводе — колонка `S` |
| `pN`, `pS` | доли несинонимных и синонимных замен на сайт | `Nd / N_sites`, `Sd / S_sites` |
| `dN`, `dS` | эволюционные расстояния с поправкой Джукса — Кантора | из `pN`, `pS` |

Индекс `d` в `Nd` и `Sd` обозначает наблюдаемые различия (differences) между
последовательностью и зародышевой линией, в отличие от `N_sites` и `S_sites` —
числа позиций, в которых такие различия возможны.

### Порядок расчёта

1. Для каждого кодона `ref_codon` определяется число синонимных (`s`) и
   несинонимных (`n`) сайтов, `n + s = 3`. В каждой из трёх позиций кодона
   рассматриваются три возможные замены; доля замен, не изменяющих
   аминокислоту, составляет вклад в `s`. Например, для `GCC` (Ala) синонимна
   вся третья позиция, `s = 1`; для `ATG` (Met) любая замена изменяет
   аминокислоту, `s = 0`.
2. Суммирование по всем мутациям области: `N_sites = Σn`, `S_sites = Σs`.
3. Доли замен: `pN = Nd / N_sites`, `pS = Sd / S_sites`.
4. Эволюционные расстояния: `dN = −(3/4)·ln(1 − (4/3)·pN)`, аналогично `dS`.
5. Отношение `dN/dS`.

Стоп-кодоны и кодоны с неопределёнными нуклеотидами при подсчёте сайтов
исключаются.

Значение `dN/dS > 1` указывает на положительный отбор, ожидаемый в областях
CDR, определяющих связывание антигена. Значение `dN/dS < 1` указывает на
очищающий отбор, ожидаемый в каркасных областях FR, где замены нарушают
структуру домена.

## Верификация

Подсчёт сайтов сверен с референсными значениями: `ATG` — `s = 0`, `TTT` —
`s = 1/3`, `GCC` — `s = 1`, `CGA` — `s = 4/3`. Итоговые значения dN/dS сверены с
независимым расчётом на контрольном наборе: при `Nd = 4`, `Sd = 1`, кодон `GCC`
получено `dN/dS = 2.457`; при `Nd = 1`, `Sd = 2` — `dN/dS = 0.114`.

## Ограничения

Величины `N_sites` и `S_sites` вычисляются по кодонам `ref_codon` наблюдаемых
мутаций, поскольку полная последовательность зародышевой линии в `mutations.tsv`
отсутствует. Это оценка нуклеотидного состава по мутировавшим кодонам, а не по
всему V-гену; при неравномерном распределении соматического гипермутагенеза
(мотивы WRCY/RGYW) оценка смещена. Для несмещённого расчёта требуются все кодоны
зародышевой линии; их извлечение из `references/all_V.fasta` по полю `v_gene`
возможно, однако границы областей FR/CDR в зародышевой последовательности без
IgBLAST не определены, что исключает пораздельный расчёт. Для сопоставления
областей между собой (CDR относительно FR) используемая оценка применима.

## Требование к входным данным

Расчёт dN/dS требует наличия синонимных замен в `mutations.tsv` как знаменателя
`dS`. В исходной версии `analyze_mutations.py` отбор мутаций выполнялся по
условию `q_aa != s_aa`, то есть регистрировались только замены, изменяющие
аминокислоту; синонимные замены в таблицу не попадали, а поле `is_silent`
принимало значение `no` для всех строк.

Внесённое изменение переводит условие отбора с аминокислотного на кодонное:

```python
if q_codon != s_codon:
    is_silent = (q_aa == s_aa)
```

Контрольная проверка на последовательностях `GCCACTGGTTTT` (зародышевая линия) и
`GCAAGTGGTTTC` (наблюдаемая), содержащих две синонимные замены (`GCC→GCA`,
`TTT→TTC`) и одну несинонимную (`ACT→AGT`):

```
до изменения    всего=1  синонимных=0  несинонимных=1
после изменения всего=3  синонимных=2  несинонимных=1
```

Следствия изменения: `mutations.tsv` дополняется строками синонимных замен;
`mutations_summary.tsv` не изменяется, поскольку счётчики в нём фильтруются
условием `not m['is_silent']`.
