# mutation_stats

Где мутации (в каких FR/CDR) и соотношение **dN/dS** — по выходу шага Дениса
(`scripts/analyze_mutations/analyze_mutations.py` → `mutations.tsv`).

## Вход / выход

| | формат | путь |
|---|---|---|
| вход | `mutations.tsv` (или папка `mutation_tables/` — тогда рекурсивно все) | выход `analyze_mutations.sh` |
| выход | таблица в консоль; `--tsv` — файлом | — |

## Запуск

```bash
python mutation_stats.py mutation_tables/                  # сводка по всем кладам
python mutation_stats.py mutation_tables/ --per-clade      # + отчёт по каждой кладе
python mutation_stats.py mutation_tables/clade1/mutations.tsv
python mutation_stats.py mutation_tables/ --tsv stats.tsv  # сохранить таблицу
```

Зависимостей нет — чистый Python 3.9+ (stdlib).

## Пример вывода

```
регион                  мутаций    доля     N     S    N/S    dN/dS
------------------------------------------------------------------------------
FR1-IMGT                      3   37.5%     1     2   0.50    0.114
CDR1-IMGT                     5   62.5%     4     1   4.00    2.457  ← CDR
------------------------------------------------------------------------------
ВСЕ CDR                       5   62.5%     4     1   4.00    2.457
ВСЕ FR                        3   37.5%     1     2   0.50    0.114
```

- `N` — несинонимные замены (меняют аминокислоту), `S` — синонимные
- `dN/dS > 1` — положительный отбор, ожидается в **CDR** (петли связывают антиген,
  выгодные замены закрепляются)
- `dN/dS < 1` — очищающий отбор, ожидается в **FR** (каркас держит структуру,
  замены вредны и отсеиваются)

## ⚠ Блокер: dN/dS сейчас не считается (баг в analyze_mutations.py)

**`mutations.tsv` не содержит синонимных мутаций вообще**, поэтому знаменатель
dS всегда пуст и dN/dS посчитать нельзя. Скрипт это детектирует и говорит прямо,
а не печатает бессмысленное число.

Причина — в `scripts/analyze_mutations/analyze_mutations.py`, строка ~318:

```python
if q_aa != s_aa:                                              # ← только если АК изменилась
    is_silent = (q_aa == '*' and s_aa == '*') or (q_aa == s_aa)
```

Синонимная замена (нуклеотид другой, аминокислота та же) не проходит условие
`q_aa != s_aa` и не записывается. А `is_silent` внутри этой ветки вычисляется
из `q_aa == s_aa`, которое здесь **всегда `False` по построению** — то есть
колонка `is_silent` в файле всегда `'no'`.

Проверено на реальном коде (germline `GCCACTGGT` → observed `GCAAGTGGT`,
где `GCC→GCA` синонимная (Ala→Ala) и `ACT→AGT` несинонимная (Thr→Ser)):

```
ДО фикса:    1 мутаций          ← синонимная потеряна
    act->agt  T->S  is_silent=False
ПОСЛЕ фикса: 2 мутаций
    gcc->gca  A->A  is_silent=True    ← появилась
    act->agt  T->S  is_silent=False
```

После фикса `mutation_stats.py` посчитает dN/dS автоматически — менять его не нужно.

Побочный эффект фикса (ожидаемый и правильный): `mutations.tsv` станет длиннее —
в нём появятся синонимные строки. `mutations_summary.tsv` не изменится: там
счётчики уже фильтруются через `if not m['is_silent']`.

## Как считается dN/dS

Метод **Nei–Gojobori** с коррекцией Jukes–Cantor:

1. Для каждого `ref_codon` считаются сайты: сколько из 3 возможных замен в каждой
   позиции синонимны. Например `GCC` (Ala) → вся 3-я позиция синонимна → `s=1`, `n=2`;
   `ATG` (Met) → любая замена меняет АК → `s=0`, `n=3`.
2. `N_sites = Σn`, `S_sites = Σs` по всем наблюдаемым мутациям региона.
3. `pN = Nd/N_sites`, `pS = Sd/S_sites`.
4. `dN = -¾·ln(1 - ⁴⁄₃·pN)`, `dS` аналогично (Jukes–Cantor).
5. `dN/dS = dN/dS`.

Подсчёт сайтов сверен с учебными значениями (`ATG`→0, `TTT`→⅓, `GCC`→1, `CGA`→⁴⁄₃),
итоговые dN/dS — с независимым ручным расчётом.

### Ограничение оценки сайтов

`N_sites`/`S_sites` считаются по `ref_codon` **наблюдаемых мутаций** — полной
последовательности гермлайна в `mutations.tsv` нет. Это оценка композиции по
мутировавшим кодонам, а не по всему V-гену; при неравномерном SHM
(hotspot-мотивы WRCY/RGYW) она смещена. Для строгого dN/dS нужны все кодоны
гермлайна — их пришлось бы брать из `references/all_V.fasta` по `v_gene`, но
границы FR/CDR в гермлайне без IgBLAST неизвестны, поэтому per-region расчёт так
не сделать. Для сравнения регионов между собой (CDR vs FR — что и нужно
биологически) текущая оценка работает.

Прочерк `—` в колонке dN/dS означает, что посчитать нельзя: нет синонимных,
нет сайтов, или насыщение (`pS ≥ ¾`, Jukes–Cantor не определён).
