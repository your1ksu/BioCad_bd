# SHM Hotspots and Coldspots Finder

Инструмент для поиска и анализа горячих (Hotspots) и холодных (Coldspots) точек соматического гипермутирования (SHM) в ДНК-последовательностях вариабельных доменов антител (генов IGHV).

## Функционал программы

Программа принимает на вход файлы в формате FASTA (включая выровненные последовательности с гэпами) и сканирует их на наличие биологических мотивов по правилам IUPAC. Выводит JSON файл с анализом.

### Искомые мотивы
*   Горячие точки (Hotspots): RGYW, WRCY, TAA.
*   Холодные точки (Coldspots): SYC, GGR.

### Особенности алгоритма
*   Учёт гэпов выравнивания (Gap-aware matching): Скрипт распознает мотивы сквозь дефисы (-), сохраняя оригинальные координаты выравнивания (нумерацию IMGT).
*   Флаг gapped: Автоматически помечает разорванные дефисами мотивы как gapped: true для последующей фильтрации технического шума.
*   Посимвольный сдвиг: Учитывает перекрывающиеся и вложенные друг в друга мотивы.
*   Формат JSON: Выгружает результаты в структурированный файл для автоматического анализа в Python/R.

## Структура проекта

*   SHM_cold_spots_hot_spots.py — основной аналитический скрипт на Python.
*   test_SHM.py — автономный скрипт для запуска тестов (без внешних библиотек).
*   run_pipeline.sh — Bash-скрипт для автоматической установки окружения и запуска.

## Инструкция по запуску (Автоматический режим)

Bash-скрипт проверяет наличие Conda в системе. Если она отсутствует, он самостоятельно скачивает и устанавливает Miniconda, создает изолированное окружение с Python 3.11 и запускает расчеты.

1.  Перейдите в папку проекта в терминале.
2.  Сделайте скрипт исполняемым:
    ```bash
    chmod +x run_pipeline.sh
    ```
3.  Запустите конвейер, указав пути через ключи -i (входной FASTA) и -o (выходной JSON):
    ```bash
    ./run_pipeline.sh -i ~/Документы/MACSE/v/IGHV1-2_04_aligned.fasta -o ~/Документы/SHM/report.json
    ```

## Ручной запуск и тестирование

Требуется установленный Python 3.10+. Скрипт принимает как одиночные файлы, так и папки.

### Запуск анализа напрямую
```bash
python3 SHM_cold_spots_hot_spots.py --input /путь/к/файлу.fasta --output /путь/к/отчету.json
```

### Запуск тестов
```bash
python3 test_SHM.py
```

## Структура выходного JSON-файла

Выходной файл представляет собой массив объектов (по одному на каждую последовательность из FASTA).

### Пример структуры данных:
```json
[
    {
        "sequence_id": [
            "AACTGGTGTTATCGGT-1_contig_2"
        ],
        "description": "AACTGGTGTTATCGGT-1_contig_2 shm_hotspots=27",
        "total_hotspots": 27,
        "total_coldspots": 5,
        "matches": [
            {
                "type": "hotspot",
                "motif_name": "RGYW",
                "sequence": "AGCT",
                "start_index": 7,
                "end_index": 11,
                "gapped": false
            },
            {
                "type": "coldspot",
                "motif_name": "SYC",
                "sequence": "GCT",
                "start_index": 45,
                "end_index": 52,
                "gapped": true
            }
        ]
    }
]
```

### Описание полей:
*   sequence_id: Идентификатор гена (контига).
*   total_hotspots / total_coldspots: Суммарное количество найденных точек каждого типа в данном гене.
*   matches: Список найденных сайтов.
    *   type: Категория сайта (hotspot или coldspot).
    *   motif_name: Паттерн мутации.
    *   sequence: Нуклеотиды ДНК, совпавшие с мотивом.
    *   start_index / end_index: Координаты начала и конца мотива в исходном выровненном файле (индексация от 0).
    *   gapped: Маркер true/false, указывающий на наличие дефисов внутри мотива.
