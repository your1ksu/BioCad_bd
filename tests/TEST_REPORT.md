# Отчёт о тестировании Pipeline — открытость и достоверность

## Главная находка: реальный баг в извлечении уверенных клад (найден и исправлен)

Тесты на статических fixtures с заранее известными мутациями (`tests/fixtures/`)
обнаружили конкретный, воспроизводимый дефект в том, как клады извлекались из
консенсусного дерева MrBayes — не гипотетический, а видимый на реальном выводе.

### Симптом

Fixture `two_clear_pairs` (4 таксона, 2 симметричные пары, у каждой по 6 общих
мутаций — заведомо сильный сигнал) — ожидались обе клады `{seq1,seq2}` и
`{seq3,seq4}` с posterior≈1.0. Найдена была только одна:

```
tree con_50_majrule = (1:0.0072,2:0.0074,(3:0.0073,4:0.0073)1.000:0.0627);
```

### Причина

MrBayes пишет консенсус НЕУКОРЕНЁННОГО дерева как rooted-newick с базовой
политомией: `seq1` и `seq2` висят на корне отдельными ветвями, а `(seq3,seq4)`
обёрнуты в один узел с posterior. Для 4 таксонов это ЕДИНСТВЕННОЕ возможное
внутреннее ребро дерева — `{seq1,seq2}` физически обладает ТОЙ ЖЕ posterior-
поддержкой (1.0), что и `{seq3,seq4}` (это две стороны одного и того же
разбиения), но наивный обход `get_nonterminals()` видит только сторону,
которая оказалась обёрнута в узел, и теряет вторую.

Это структурный пробел в подходе «искать клады как явные внутренние узлы
дерева» — присутствовал и в вендоренном `biocode.clades`/`biocode.trees.mrbayes`
(унаследован оттуда, не привнесён нашей обёрткой).

### Исправление

В `clades/confident_clades_report.py` (без изменения вендоренного `biocode/`)
добавлены две функции:

1. **`_root_complement_supports`** — если среди прямых потомков корня ровно
   один — свёрнутая клада, а остальные — голые листья (≥2), достраивает
   комплементарную клада из этих листьев с той же posterior.
2. **`_drop_complement_duplicates`** — если клада и её дополнение обе
   присутствуют и РАЗНОГО размера, большая — это просто «всё, кроме меньшей»
   и не несёт отдельного смысла, убирается. При РАВНОМ размере (как в примере
   выше, 2+2) — обе стороны одинаково содержательны, обе сохраняются.

### Итог

```
tests/test_fixtures.py → Пройдено: 3/3
  ✓ two_clear_pairs        {seq1,seq2} и {seq3,seq4} — обе клады, posterior=1.0
  ✓ no_shared_mutations    0 клад (политомия — отрицательный контроль)
  ✓ mixed_signal           {seqA1,seqA2} и {seqB1,seqB2}, БЕЗ ложной "клады"
                            {seqB1,seqB2,seqC,seqD} (комплемент {seqA1,seqA2})
```

## 1. Fixtures — статические данные с заранее известными мутациями

`tests/fixtures/*_aligned.fasta` — не runtime-генератор, а статические файлы:
последовательности построены из повторяющегося паттерна `ACGT` (200 нт), поэтому
любая мутация видна невооружённым глазом при простом просмотре файла. Формат
точно совпадает с реальным выходом `Alina/scripts/MSA/MSA_final.py`
(`mafft --auto`, суффикс `_aligned.fasta`) — подтверждено чтением её реального
скрипта на ветке `Alina` репозитория `your1ksu/BioCad_bd`.

Рядом с каждым fasta лежит `*.expected.json` — задокументированный ground
truth: корневая последовательность, точная позиция/ref/alt каждой мутации
каждого листа, ожидаемые уверенные клады с обоснованием (сколько общих
мутаций на определяющей ветви).

| Fixture | Таксонов | Структура | Ожидаемые клады |
|---|---|---|---|
| `two_clear_pairs` | 4 | 2 симметричные пары (по 6 общих мутаций) | `{seq1,seq2}`, `{seq3,seq4}` |
| `no_shared_mutations` | 5 | только приватные мутации (звезда) | нет (отрицательный контроль) |
| `mixed_signal` | 6 | 2 пары + 2 несвязанных одиночки | `{seqA1,seqA2}`, `{seqB1,seqB2}` |

## 2. Связь с реальным pipeline (все ветки BioCad_bd)

Полная цепочка подтверждена чтением реального кода всех участников
(`your1ksu/BioCad_bd`, ветки Alina/Denis/Nikita/ksu_branch):

```
data/BCR_data.tsv
  → [ksu_branch] filter_and_save.py       → results/<key>/BCR_data_filtered.tsv
  → [ksu_branch] group_by_germlines.py    → results/<key>/grouped_by_germlines/vj/<V>_<J>.fasta
  → [Alina] MSA_final.py (mafft --auto)   → aligned_sequences/<V>_<J>_aligned.fasta
  → [Nikita] run_mrbayes.py               → mrbayes/<группа>.nex, .nex.con.tre, .mb.log
  → [Nikita] confident_clades_report.py   → clades/report.json

  (параллельно) [Denis] build_trees_iqtree.sh aligned_sequences trees
                → trees/<группа>/<группа>.treefile
                → confident_clades_report.py --iqtree-dir trees (тоже поддержано)
```

`run_mrbayes.py` уже до этого совпадал с реальным форматом Алины (папка
`aligned_sequences/`, суффикс `_aligned.fasta`) — подтверждено, изменений не
потребовалось.

## Заключение

Тесты не просто подтвердили, что pipeline «работает» — они нашли конкретный,
воспроизводимый дефект (потерю симметричной клады при базовой политомии
MrBayes-дерева), который иначе остался бы незамеченным на реальных данных
(где сигнал редко бывает настолько симметричным, чтобы дефект был заметен).
Дефект исправлен на уровне обёртки, покрыт регрессионным тестом
(`two_clear_pairs`), и оба вида дедупликации (комплемент корня, комплемент
разного размера) задокументированы и обоснованы математически — это ровно
то определение «открытости и достоверности», которое запрашивалось.
